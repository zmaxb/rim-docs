В РИМ можно выделить два уровня интеграции с мессенджерами.

**Интеграционный механизм РИМ** — это общая подсистема, которая связывает мессенджеры с внутренней моделью обработки событий в 1С.

**Интеграция мессенджера** — это реализация подключения к отдельному мессенджеру, например Telegram или VK.

Интеграционный механизм скрывает особенности API конкретных мессенджеров и позволяет прикладной логике бота работать с сообщениями, командами и событиями через единый подход.

В рамках этого раздела под **РИМ** обычно подразумевается именно интеграционный механизм. Если речь идёт о подключении к конкретному мессенджеру, используется термин **интеграция мессенджера**.

## ❓ Зачем нужны интеграции мессенджеров

У каждого мессенджера свой API, свои форматы событий, правила авторизации и способы отправки сообщений.

Без общего слоя логика бота была бы напрямую связана с конкретной платформой: Telegram, VK или другим мессенджером. Это усложняло бы поддержку, расширение и перенос ботов между разными мессенджерами.

РИМ решает эту задачу через интеграционный механизм. Интеграция мессенджера получает данные от конкретного мессенджера, приводит их к внутренней модели РИМ и передаёт дальше в общий механизм обработки.

## 🔌 За что отвечает интеграция мессенджера

Интеграция мессенджера отвечает за:

* подключение к API мессенджера;
* получение входящих событий;
* преобразование данных мессенджера во внутренние объекты РИМ;
* отправку исходящих сообщений;
* описание, применение и проверку настроек подключения.

## ⚙️ Общий принцип работы

В упрощённом виде работа интеграционного механизма выглядит так:

```text
Пользователь
    ↓
Мессенджер
    ↓
РИМ: интеграция мессенджера
    ↓
РИМ: общий механизм обработки
    ↓
Бот / прикладная логика
    ↓
РИМ: общий механизм обработки
    ↓
РИМ: интеграция мессенджера
    ↓
Мессенджер
    ↓
Пользователь
```

1. Пользователь отправляет сообщение в мессенджере.
2. Интеграция мессенджера получает входящее событие от мессенджера.
3. Интеграция приводит данные события к внутренней модели РИМ.
4. Событие передаётся в общий механизм обработки.
5. Общий механизм обработки определяет нужного бота и вызывает его прикладную логику.
6. Бот обрабатывает событие и формирует результат.
7. Результат возвращается в общий механизм обработки РИМ.
8. Общий механизм обработки передаёт результат интеграции мессенджера.
9. Интеграция мессенджера отправляет сообщение обратно в мессенджер.

## 📌 Варианты интеграций мессенджеров

В РИМ могут использоваться разные интеграции мессенджеров, например:

* Telegram;
* VK;
* другие мессенджеры, для которых реализована поддержка через интеграционный механизм РИМ.

РИМ не ограничен только Telegram и VK. Другой мессенджер может быть подключён как отдельная интеграция, если для него реализован адаптер, соответствующий общему интерфейсу интеграционного механизма РИМ.

За счёт этого прикладная логика бота может работать с разными мессенджерами единообразно: получать входящие события, обрабатывать сообщения и отправлять ответы через общий механизм, не привязываясь напрямую к API конкретного мессенджера.
